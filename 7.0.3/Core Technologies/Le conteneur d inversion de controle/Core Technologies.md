# __*Core Technologies*__

Cette partie de la documentation couvre toutes les technologies qui sont essentielles au framework _Spring_.

La première d'entre elles est le conteneur d'inversion de contrôle de _Spring_ (IoC). Un traitement approfondi du conteneur d'inversion de contrôle de _Spring_ est ensuite présenté, suivi d'une explication des technologies de la programmation orientée aspect (AOP) de _Spring_. Le framework _Spring_ dispose de son propre framework AOP, lequel est conceptuellement facile à comprendre et couvre environ 80 % des besoins AOP typiques en entreprise Java.

L'étude de l'intégration de _Spring_ avec AspectJ (actuellement le plus riche en fonctionnalités et certainement l'implémentation AOP la plus mature dans un écosystème Java d'entreprise) est également fournie.

Le traitement AOT peut être utilisé pour optimiser vos applications à l'avance. Il est typiquement utilisé pour les déploiements d'images natives avec GraalVM.
